# Google Keep to Markdown Exporter

A lightweight, dependency-free Python script that bulk-converts Google Keep notes (exported via Google Docs) into cleanly organized, individual Markdown (`.md`) files.

## ⚠️ The Problem It Solves
When you export Google Keep notes to a Google Doc and download it as a plain text (`.txt`) file, Google introduces formatting quirks:
1. Standard line breaks *inside* a note become 3 hidden line breaks.
2. The gaps *between* separate notes become 5 hidden line breaks.
3. Keep sometimes injects artifact tags like ``.

Standard text splitters will chop your individual notes into pieces. This script acts as a smart parser that understands Keep's specific export structure, safely isolating your notes and naming them based on their content.

## ✨ Features
* **Smart Splitting:** Ignored internal paragraph spacing and only splits notes based on 4+ consecutive line breaks.
* **Auto-Titling:** Uses the exact first line of your note as the `.md` filename, preserving natural spaces.
* **APFS/OS Safe:** Automatically removes illegal characters (`/`, `?`, `<`, `>`, `\`, `:`, `*`, `|`, `"`).
* **Length Limit Protection:** macOS APFS limits filenames to 255 characters. This script safely truncates massive one-liner notes at 200 characters and injects the full, uncut line into the body of the file so no data is lost.
* **Artifact Cleanup:** Automatically detects and removes `` tags generated by the Google Docs export.
* **Batch Processing:** Drop as many `.txt` files as you want into the target folder; the script will process them all and build a matching directory structure.

## 🚀 How to Use

### 1. Export Your Notes
1. In Google Keep, select the notes you want and choose **Copy to Google Docs**.
2. Open that Google Doc, go to **File > Download > Plain text (.txt)**.
3. (Optional) Repeat for different categories of notes, saving multiple `.txt` files.

### 2. Setup Your Folders
Create a working directory. Inside it, place your downloaded `.txt` files into a folder named `Keep_Exports`. Place the Python script right next to it.
