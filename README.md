# Google Keep to Markdown Exporter: The Complete Guide

A lightweight, dependency-free Python script that bulk-converts Google Keep notes (exported via Google Docs) into cleanly organized, individual Markdown (`.md`) files.

## ⚠️ The Problem It Solves

When you export Google Keep notes to a Google Doc and download it as a plain text (`.txt`) file, Google introduces formatting quirks:
1. Standard line breaks *inside* a note become 3 hidden line breaks.
2. The gaps *between* separate notes become 5 hidden line breaks.
3. Keep sometimes injects artifact tags like ``.

Standard text splitters will chop your individual notes into pieces. This script acts as a smart parser that understands Keep's specific export structure, safely isolating your notes and naming them based on their content.

## ✨ Features

* **Smart Splitting:** Ignores internal paragraph spacing and only splits notes based on 4+ consecutive line breaks.
* **Auto-Titling:** Uses the exact first line of your note as the `.md` filename, preserving natural spaces.
* **APFS/OS Safe:** Automatically removes illegal characters (`/`, `?`, `<`, `>`, `\`, `:`, `*`, `|`, `"`).
* **Length Limit Protection:** macOS APFS limits filenames to 255 characters. This script safely truncates massive one-liner notes at 200 characters and injects the full, uncut line into the body of the file so no data is lost.
* **Artifact Cleanup:** Automatically detects and removes `` tags generated by the Google Docs export.
* **Batch Processing:** Drop as many `.txt` files as you want into the target folder; the script will process them all and build a matching directory structure.

## 🚀 The Complete Process

### Phase 1: Exporting from Google Keep

Because Google Takeout exports Keep notes in a messy HTML/JSON format, the cleanest method is to route them through Google Docs.

1. **Select Your Notes:** Open Google Keep on your computer. Hover over a note and click the checkmark to select it. You can select multiple notes, or use `Ctrl+A` / `Cmd+A` to select everything.
2. **Copy to Docs:** Click the three-dot menu (⋮) at the top right of the screen and select **Copy to Google Docs**.
3. **Download as Text:** Once the copy is complete, click the link to open the new Google Doc. Go to **File > Download > Plain text (.txt)**. 
4. *(Optional)* If you categorize your notes, repeat this process for each category, downloading multiple `.txt` files (e.g., `Ideas.txt`, `Journal.txt`, `Bookmarks.txt`).

### Phase 2: Setting up Your Local Environment

To keep things organized, we need to set up a specific folder structure on your computer.

1. Create a main working folder anywhere on your Mac (e.g., `Keep_Migration`).
2. Inside that folder, create a subfolder exactly named `Keep_Exports`.
3. Move all of your downloaded `.txt` files into the `Keep_Exports` folder.

Your folder structure should look exactly like this:
```text
/Keep_Migration
  └── /Keep_Exports
      ├── Ideas.txt
      └── Journal.txt
```

### Phase 3: The Python Exporter Script

**Create the Script:**
1. Open a plain text editor (like TextEdit or VS Code).
2. Paste the code below.
3. Save it as `batch_exporter.py` inside your main `/Keep_Migration` folder (right next to the `Keep_Exports` folder, not inside it).

```python
import os
import re
import glob

def process_note_batches(target_folder):
    # 1. Verify the target folder exists
    if not os.path.exists(target_folder):
        print(f"Error: The folder '{target_folder}' does not exist.")
        return

    # 2. Find all .txt files
    search_pattern = os.path.join(target_folder, '*.txt')
    text_files = glob.glob(search_pattern)

    if not text_files:
        print(f"No .txt files found in '{target_folder}'.")
        return

    print(f"Found {len(text_files)} files to process. Starting export...\n")

    # 3. Process each .txt file individually
    for filepath in text_files:
        base_filename = os.path.basename(filepath)
        folder_name = os.path.splitext(base_filename)[0]
        
        # Create a matching output directory
        output_dir = os.path.join(target_folder, folder_name)
        os.makedirs(output_dir, exist_ok=True)
        
        with open(filepath, 'r', encoding='utf-8') as f:
            content = f.read()
            
        # Split ONLY on 4 or more consecutive line breaks to bypass internal paragraph spacing
        notes = re.split(r'\n(?:\s*\n){3,}', content)
        
        saved_count = 0
        for note in notes:
            note = note.strip()
            
            # Clean out any artifacts from Google Docs
            note = re.sub(r'^\\s*', '', note).strip()
            
            if not note:
                continue
                
            # Separate the title (first line) from the body
            lines = note.split('\n')
            first_line = lines[0].strip()
            body_text = '\n'.join(lines[1:]).strip()
            
            # Clean the filename of illegal OS characters
            safe_title = re.sub(r'[\/\?<>\\:\*\|"]', '', first_line).strip()
            
            # Enforce a safe character limit for macOS (APFS limit is 255)
            # If too long, truncate the title but save the full line into the body
            if len(safe_title) > 200:
                safe_title = safe_title[:200].strip() + "..."
                body_text = f"{first_line}\n\n{body_text}".strip()
            
            # Fallback for completely empty titles
            if not safe_title:
                safe_title = f"Untitled_Note_{saved_count + 1}"
                
            # Write the final .md file
            out_filepath = os.path.join(output_dir, f"{safe_title}.md")
            with open(out_filepath, 'w', encoding='utf-8') as out_f:
                if body_text:
                    out_f.write(body_text + '\n')
                    
            saved_count += 1
            
        print(f"✅ Processed '{base_filename}': Saved {saved_count} notes into '/{folder_name}'")

    print("\nAll files have been successfully processed!")

# --- Execution ---
BATCH_FOLDER = "Keep_Exports"
process_note_batches(BATCH_FOLDER)
```

### Phase 4: Execution

With your files in place and your script saved, you are ready to process your notes.

1. Open your Mac's **Terminal**.
2. Navigate to your working folder using the `cd` command. For example:
   ```bash
   cd Desktop/Keep_Migration
   ```
3. Run the python script:
   ```bash
   python3 batch_exporter.py
   ```

**The Output:**
The script will instantly create folders matching your original `.txt` filenames and populate them with your freshly formatted Markdown notes.

```text
/Keep_Migration
  ├── batch_exporter.py
  └── /Keep_Exports
      ├── Ideas.txt
      ├── /Ideas                 <-- New Folder!
      │   ├── App_concept.md
      │   └── Grocery_list.md
      ├── Journal.txt
      └── /Journal               <-- New Folder!
          ├── May_15_entry.md
          └── May_16_entry.md
```

## 🛠 Prerequisites
* Python 3.x
* No external libraries required. It relies entirely on standard Python modules (`os`, `re`, `glob`).
